# marimo ノートブック作成ルール

解説を marimo ノートブックとしてまとめるときのルールとテンプレート。
解説内容そのもののルールは `explanation-rules.md` に従うこと。

## 基本ルール

- ファイル配置先はプロジェクト設定に従う（SKILL.md 参照）
- インタラクティブコントロール（`mo.ui.slider` / `mo.ui.dropdown` 等）を積極的に使う
- グラフタイトル・軸ラベルは日本語で書く
- 日本語フォントを必ず設定する（フォント名はプロジェクト設定を参照）

```python
plt.rcParams["font.family"] = "Hiragino Sans"  # macOS の場合
plt.rcParams["axes.unicode_minus"] = False
```

## 注意：`mo.md()` 内の LaTeX バックスラッシュ

`mo.md("""...""")` の中で `\frac`, `\alpha`, `\top`, `\nabla`, `\varepsilon`, `\right` などを
**単一バックスラッシュで書かない**こと。Python のエスケープシーケンス
（`\f`=改ページ, `\a`=ベル, `\t`=タブ, `\n`=改行, `\v`=垂直タブ, `\r`=復帰）として解釈され、
保存を繰り返すうちにファイルが制御文字で壊れる。

- 必ず raw 文字列 `mo.md(r"""...""")` を使うか、`\\frac` のように二重バックスラッシュで書く

## グラフのレイアウト

- 補助グラフ（関数曲線など）を `inset_axes` でヒートマップ等の上に重ねない。
  独立したサブプロット（`plt.subplots(1, N)` + `width_ratios`）として横に並べる
- `plt.tight_layout()` を呼び、`suptitle` は `y=1.02` 程度で重なりを避ける

## セル構成テンプレート

```python
# セル1：タイトルと概要
mo.md(r"""
# <演算名>：何を表現しているのか？

<演算名> は〜に使われる重要な演算です。
「何を計算しているのか」だけでなく「どんな現象を表すか」を視覚的に理解しましょう。
""")

# セル2：数学的定義
mo.md(r"""
## 計算方法

$$
<数式>
$$

## 何を表現しているか？

> **「<直感的な一言説明>」**

- 条件A → <何が起きるか>
- 条件B → <何が起きるか>
""")

# セル3：インポートとフォント設定
import numpy as np
import matplotlib.pyplot as plt

plt.rcParams["font.family"] = "Hiragino Sans"
plt.rcParams["axes.unicode_minus"] = False

# セル4：インタラクティブコントロール（スライダー等）
slider = mo.ui.slider(start=..., stop=..., step=..., value=..., label="...")

# セル5：グラフ描画（slider.value を使用）
fig, ax = plt.subplots(...)
# ... グラフ描画 ...
ax.set_title("日本語タイトル", fontsize=12)
ax.set_xlabel("横軸ラベル（単位）")
ax.set_ylabel("縦軸ラベル（単位）")
fig

# セル6：インタラクション説明
mo.md(r"""
> **操作ガイド**：スライダーを動かすと〜が変わります。
> - 値A のとき：<観察できること>
> - 値B のとき：<観察できること>
""")

# セル7：まとめ表
mo.md(r"""
## まとめ

| 性質 | <演算A> | <演算B> |
|:---:|:---:|:---:|
| 入力 | ... | ... |
| 出力 | ... | ... |
| 表現すること | ... | ... |
| コード | `...` | `...` |
| 応用例 | ... | ... |
""")
```
