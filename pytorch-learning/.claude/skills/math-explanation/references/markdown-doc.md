# Markdown 解説ドキュメント作成ルール

解説を Markdown ドキュメントとしてまとめるときのルールとテンプレート
（例：チュートリアルの `.py` に対応する解説 Markdown）。
解説内容そのもののルール・途中計算フォーマット・チェックリストは `explanation-rules.md` に従うこと。

## ファイル配置

- 解説対象の `.py` と **同じディレクトリ** に置く
- ファイル名はプロジェクト設定に従う（SKILL.md 参照。既定: `math_explanation.md`）

## セクション構成の原則

各演算のセクションは次の順で書く：

1. **コード提示**（解説対象のコード）
2. **数学的定義**（LaTeX + 読み方の日本語説明 + ASCII 図）
3. **入力値の明示**（行列の実際の値を先に示す）
4. **全要素の途中計算**（省略せず行ごとにまとめる）
5. **結果行列**（+ 読み取れる数学的性質）
6. **使いどころ**（実用例の表）
