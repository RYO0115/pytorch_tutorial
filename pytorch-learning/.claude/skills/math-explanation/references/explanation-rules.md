# 数学・行列演算・数式解説の共通ルール

数学概念・行列演算・線形代数を解説するときの、成果物の形式によらない共通ルール。

## 言語・表記

- 説明文はすべて **日本語** で書く
- 数式は LaTeX 記法（`$$...$$` / `$...$`）を使う
- インラインコードはバッククォート（`` `code` ``）で囲む
- 行列・ベクトルの添字は 0 始まりで統一する（Python の慣習に合わせる）
- コード例はプロジェクト設定のフレームワークで示す（SKILL.md 参照）

## 解説の構成順序（必須）

1. **計算方法**（数式 + コード）
2. **何を表現しているか**（直感・現象の意味を1文で）
3. **具体的な計算例**（数値を使って全要素を手計算）
4. **視覚化**（グラフ・図解）
5. **使いどころ**（実応用との対応）

## 途中計算の書き方

途中計算は「一部だけ示して省略」しない。**全要素**を行ごとにまとめて書く。

### 行列積

```
C[i,j]:  A[i,0]×B[0,j] + A[i,1]×B[1,j] + ...  =  <各積の展開>  =  <合計>

例）
C[0,0]:  1×2 + 3×4  =  2 + 12  =  14
```

### 要素積

```
位置 (i,j):  A[i,j] × B[i,j]  =  <値> × <値>  =  <結果>  →  Z[i,j] = <結果>

例）
位置 (0,0):  A[0,0] × A[0,0]  =  1 × 1  =  1  →  Z[0,0] = 1
```

### 外積

```
outer[i,j] = a[i] × b[j]

例）
outer[0,1] = a[0] × b[1] = 2.0 × 1.5 = 3.0
```

## 図示ルール（ASCII アート）

- 「どの行とどの列が掛け合わさるか」を ASCII アートで視覚化する
- 要素の対応は `×` と `↓` と `→` で流れを示す
- 結果への「流れ」は `→ Z[i,j] = 値` の形で明記する
