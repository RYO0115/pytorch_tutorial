# チュートリアル解説 Markdown 作成ルール

PyTorch チュートリアルの Python ファイルに対応する解説 Markdown を作成するための規則とテンプレートです。

---

## 作成ルール

### ファイル配置

- 解説対象の `.py` と **同じディレクトリ** に置く
- ファイル名: `math_explanation.md`（固定）

### 言語・表記

- 説明文はすべて **日本語** で書く
- 数式は LaTeX 記法（`$$...$$`）を使う
- インラインコードはバッククォート（`` `code` ``）で囲む
- 行列・ベクトルの添字は 0 始まりで統一する（Python の慣習に合わせる）

### セクション構成の原則

1. **コード提示** → 2. **数学的定義** → 3. **入力値の明示** → 4. **全要素の途中計算** → 5. **結果行列** の順で書く
2. 途中計算は「一部だけ示して省略」しない。全要素を行ごとにまとめて書く
3. 演算ごとに使いどころ（実用例）を添える

### 図示ルール

- ASCII アートで「どの行とどの列が掛け合わさるか」を視覚化する
- 要素の対応は `×` と `↓` と `→` で流れを示す
- 結果への「流れ」は `→ Z[i,j] = 値` の形で明記する

## セクション別チェックリスト

### 行列積 (`@` / `matmul`)

- [ ] $C_{ij} = \sum_k A_{ik} B_{kj}$ の定義を記載した
- [ ] 「第 $i$ 行 × 第 $j$ 列」の ASCII 図を入れた
- [ ] 転置（`.T`）を使う場合、列が元の行と一致することを注記した
- [ ] 全要素（行数 × 列数 個）の途中計算を行ごとにまとめた
- [ ] 結果の数学的性質（対称性など）を注記した
- [ ] 実用例（全結合層、Attention など）を記載した

### 要素積 (`*` / `mul`)

- [ ] 「同じ位置 $(i,j)$ の要素どうし」という制約を明記した
- [ ] 入力行列の値を先に示した
- [ ] `位置 (i,j): A[i,j] × B[i,j] = 値 → Z[i,j] = 値` の形式で全要素を列挙した
- [ ] 行列積との shape の違いを比較表で示した
- [ ] 実用例（マスク、ゲートなど）を記載した

### べき乗 (`pow`)

- [ ] 各要素に独立して適用されることを明記した
- [ ] 要素積との等価性を示した
- [ ] 実用例（MSE損失、L2正則化など）を記載した

## 途中計算の書き方フォーマット

### 行列積の場合

```
C[i,j]:  A[i,0]×B[0,j] + A[i,1]×B[1,j] + ...  =  <各積>  =  <合計>
```

例:
```
C[0,0]:  1×1 + 0×0 + 1×1 + 1×1  =  1 + 0 + 1 + 1  =  3
```

### 要素積の場合

```
位置 (i,j):  A[i,j] × B[i,j]  =  <値> × <値>  =  <結果>   →  Z[i,j] = <結果>
```

例:
```
位置 (0,0):  A[0,0] × A[0,0]  =  1 × 1  =  1   →  Z[0,0] = 1
```
